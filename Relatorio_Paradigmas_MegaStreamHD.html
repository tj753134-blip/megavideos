Resumo

Este relatório apresenta uma análise detalhada dos paradigmas de programação aplicados no desenvolvimento do projeto MegaStreamHD, uma plataforma web interativa de streaming de filmes e séries desenvolvida em JavaScript puro (Vanilla JS). O projeto foi construído como resposta aos requisitos da disciplina de Linguagens de Programação, demonstrando a aplicação prática de três paradigmas principais: orientado a eventos, declarativo e imperativo.

O MegaStreamHD integra uma arquitetura completa com frontend público (HTML/CSS/JavaScript), painel administrativo, sistema de autenticação via Supabase e um banco de dados relacional com políticas de segurança (RLS). A análise evidencia como diferentes paradigmas coexistem e complementam-se numa aplicação moderna, oferecendo vantagens específicas em termos de interatividade, estrutura e manutenibilidade.

Palavras-chave: Paradigmas de Programação, JavaScript, Event-Driven, Declarativo, Imperativo, Aplicação Web, Frontend, Supabase.

1. Introdução

1.1 Contextualização

O projeto MegaStreamHD surge como resposta ao Projeto 9 da disciplina de Linguagens de Programação, com o objetivo de criar um sistema web interativo que demonstre a aplicação prática de diferentes paradigmas de programação. A escolha de JavaScript como linguagem principal decorre da sua natureza intrinsecamente orientada a eventos e da sua prevalência no desenvolvimento web contemporâneo.

1.2 Justificativa

As aplicações web modernas raramente seguem um único paradigma de forma isolada. O MegaStreamHD foi desenvolvido com a intenção de demonstrar como diferentes paradigmas podem coexistir harmoniosamente dentro de um único sistema, cada um contribuindo para diferentes aspectos da solução: interatividade (eventos), estrutura (declarativo) e lógica (imperativo).

1.3 Objetivos

Geral: Desenvolver uma aplicação web interativa que demonstre a aplicação prática dos paradigmas de programação orientado a eventos, declarativo e imperativo.

Específicos:

  • Implementar um sistema web com JavaScript que responda a eventos de utilizador
  • Demonstrar o uso de paradigma orientado a eventos na interatividade da aplicação
  • Exemplificar paradigma declarativo em HTML, CSS e SQL
  • Analisar paradigma imperativo na lógica interna das funções
  • Integrar autenticação e persistência de dados com Supabase

3. Análise do Sistema

3.1 Arquitetura Geral

O MegaStreamHD segue uma arquitetura cliente-servidor com separação clara de responsabilidades:

  • Frontend Público (index.html, app.js, style.css): Interface para utilizadores finais com navegação, filtros, detalhes de conteúdo e sistema de comentários.
  • Painel Administrativo (painel-gs7x2.html, admin.js, admin.css): Interface exclusiva para administradores com funcionalidades de CRUD.
  • Camada de Dados (data.js): Abstração que gerencia comunicação com Supabase, cache em memória e sincronização de dados.
  • Backend (Supabase): PostgreSQL com políticas RLS para controle de acesso e persistência de dados.

3.2 Utilizadores e Funções

Utilizador Público: Acesso de leitura a filmes, séries, comentários e informações de conteúdo. Pode fazer login, comentar e submeter pedidos.

Administrador: Acesso completo de escrita (CRUD) para conteúdo, visualização de estatísticas, gestão de pedidos e configuração do site.

4. Projeto (Design) do Sistema

4.1 Arquitetura do Frontend

O frontend segue o padrão de programação orientado a eventos com estrutura HTML/CSS/JavaScript separada:

HTML (Declarativo): Define a estrutura estática da página.

CSS (Declarativo): Especifica regras de estilo.

JavaScript (Eventos + Imperativo): Implementa handlers de eventos e lógica de processamento de dados.

4.2 Modelagem de Dados

Tabelas principais no Supabase:

7. Análise Detalhada dos Paradigmas de Programação

7.1 Paradigma Orientado a Eventos

O paradigma orientado a eventos é o predominante no MegaStreamHD. O fluxo de execução é determinado por eventos externos (cliques, hashchange, auth state changes).

<a onclick="filterContent('filmes','acao')">Ação</a>

Ex.: handlers como sup.auth.onAuthStateChange(...) executam lógica em resposta a mudanças de autenticação.

7.2 Paradigma Declarativo

Manifestado em HTML, CSS e SQL (políticas RLS declarativas).

.card { background: #1a1a1a; border-radius: 8px; }

Ex.: políticas SQL declarando regras de autorização.

7.3 Paradigma Imperativo

Presente nas funções JavaScript que especificam passo-a-passo a transformação de dados (ex.: filterContent()).
